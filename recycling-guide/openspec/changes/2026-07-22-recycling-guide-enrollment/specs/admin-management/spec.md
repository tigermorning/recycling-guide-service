## ADDED Requirements

### Requirement: 관리자는 클래스를 추가·수정·삭제할 수 있다

시스템은 관리자 권한이 있는 사용자에게 클래스 생성·수정·삭제 기능을 제공하여야 한다.

#### Scenario: 클래스 추가

- **WHEN** 관리자가 이름·설명·정원을 입력해 클래스를 추가한다
- **THEN** 시스템은 새 클래스를 생성하고 목록에 반영한다

#### Scenario: 클래스 수정/삭제

- **WHEN** 관리자가 기존 클래스를 수정하거나 삭제한다
- **THEN** 시스템은 변경을 저장하고 목록에 반영한다

### Requirement: 관리자가 아닌 사용자는 클래스 관리 기능을 사용할 수 없다

시스템은 RPC 호출과 테이블 직접 접근 양쪽 모두에서 관리자 여부를 검증하여야 한다.

#### Scenario: 비관리자의 RPC 호출 차단

- **WHEN** 관리자가 아닌 로그인 사용자가 `admin_add_class` 등 관리자 RPC를 호출한다
- **THEN** 시스템은 "관리자 권한이 필요합니다" 메시지를 반환하고 아무 것도 변경하지 않는다

#### Scenario: 비관리자의 테이블 직접 접근 차단

- **WHEN** 관리자가 아닌 로그인 사용자가 `classes` 테이블에 직접 INSERT/UPDATE를 시도한다
- **THEN** 시스템은 이를 거부한다
- **현재 상태**: 미충족 — `classes` 테이블 RLS가 `TO authenticated WITH CHECK (true)`로 되어 있어 로그인한 누구나 직접 접근이 허용됨. `tasks.md` 3.5에서 수정 예정.

### Requirement: 관리자는 클래스별 신청자 명단을 조회할 수 있다

시스템은 관리자가 특정 클래스의 신청자 이메일·상태·신청일을 조회할 수 있게 하여야 한다.

#### Scenario: 명단 조회

- **WHEN** 관리자가 클래스의 "명단" 버튼을 누른다
- **THEN** 시스템은 해당 클래스의 신청자 목록을 모달로 표시한다
- **현재 상태**: 미충족 — 버튼 클릭 시 아무 반응이 없는 버그가 있음. `tasks.md` 3.4에서 원인 조사 및 수정 예정.
