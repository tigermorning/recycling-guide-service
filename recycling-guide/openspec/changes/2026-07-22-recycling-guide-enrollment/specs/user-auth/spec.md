## ADDED Requirements

### Requirement: 사용자는 이메일과 비밀번호로 회원가입하고 로그인할 수 있다

시스템은 이메일·비밀번호 기반 회원가입과 로그인을 제공하여야 한다.

#### Scenario: 회원가입

- **WHEN** 새 사용자가 이메일과 비밀번호를 입력해 회원가입한다
- **THEN** 시스템은 계정을 생성하고 로그인 상태로 전환한다

#### Scenario: 로그인

- **WHEN** 기존 사용자가 이메일과 비밀번호로 로그인한다
- **THEN** 시스템은 세션을 생성하고 로그인 상태를 화면에 반영한다

#### Scenario: 로그아웃

- **WHEN** 로그인한 사용자가 로그아웃을 누른다
- **THEN** 시스템은 세션을 종료하고 로그인 전 화면으로 되돌린다

### Requirement: 로그인한 사용자는 자신의 신청 내역만 조회할 수 있다

시스템은 행 수준 보안(RLS)으로 사용자가 자신의 신청 내역에만 접근하도록 제한하여야 한다.

#### Scenario: 본인 신청 내역 조회

- **WHEN** 로그인한 사용자가 자신의 신청 목록을 조회한다
- **THEN** 시스템은 해당 사용자의 신청만 반환한다

#### Scenario: 타인 신청 내역 비공개

- **WHEN** 로그인한 사용자가 다른 사용자의 신청 데이터에 접근을 시도한다
- **THEN** 시스템은 이를 반환하지 않는다
