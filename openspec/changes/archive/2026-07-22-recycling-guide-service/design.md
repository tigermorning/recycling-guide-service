## Context

한국 시민들은 분리수거를 할 때 기본은 알지만 세부 기준을 모르는 경우가 많습니다. 이 문제를 해결하기 위한 품목별 분리수거 가이드 서비스를 만듭니다. MVP는 서울시 기준으로 시작하며, 향후 전국 확장을 목표로 합니다.

현재 상태: 신규 프로젝트
제약 조건: 
- 브라우저에서 동작하는 웹 서비스
- 외부 API 의존성 없음 (정적 데이터)
- 모바일/데스크톱 모두 지원

## Goals / Non-Goals

**Goals:**
- 사용자가 품목을 검색하면 서울시 기준 분리수거 방법을 한 화면에 확인할 수 있다
- 세부 품목(코팅된 종이, 충전선 등)도 검색 가능하다
- 직관적인 인터페이스로 빠른 정보 확인이 가능하다

**Non-Goals:**
- AI 사진 인식 기능
- 전국 지역별 규칙 (서울부터 시작)
- 사용자 계정/로그인
- 커뮤니티/댓글 기능
- 게이미피케이션/포인트

## Decisions

### 1. 프론트엔드 기술 선택: 순수 HTML/CSS/JS

**근거:**
- MVP 단계에서 프레임워크 불필요
- 로딩 속도 최우선 (검색 서비스의 핵심)
- 배포 용이성 (GitHub Pages 등)

**대안 검토:**
- React/Vue: 복잡도 증가, MVP 불필요
- Next.js: SSR 불필요 (정적 데이터)

### 2. 데이터 구조: JSON 파일

**근거:**
- 서울시 분리수거 기준은 고정 데이터
- API 서버 불필요
- 로컬 파일로 관리 용이

**구조:**
```json
{
  "items": [
    {
      "name": "비닐",
      "aliases": ["비닐봉지", "일회용 비닐"],
      "category": "일반쓰레기",
      "disposalMethod": "내용물 비우고 일반쓰레기 배출",
      "notes": ""
    }
  ]
}
```

### 3. 검색 알고리즘: 부분 일치 + 별명 지원

**근거:**
- 사용자가 다양한 이름으로 검색할 수 있어야 함
- "코팅된 종이" = "코팅 종이" = "코팅종이"

**방법:**
- indexOf() 기반 부분 일치
- 별명(alias) 배열로 다중 키워드 지원

### 4. 반응형 디자인: 모바일 우선

**근거:**
- 실제 사용자가 쓰레기를 들고 검색하는 상황은 모바일이 일반적
- 모바일에서 사용하기 쉬운 인터페이스

## Risks / Trade-offs

### [데이터 부정확성] → 서울시 공식 데이터 확인 후 반영

- 서울시 분리수거 기준이 변경될 수 있음
- 완성 후 서울시 환경부 공식 데이터와 대조 필요

### [검색 정확도] → 별명 데이터 추가로 보완

- 사용자가 검색하는 키워드가 다를 수 있음
- 자주 사용되는 별명을 데이터에 추가

### [성능] → 데이터량이 많아지면 문제될 수 있음

- 현재는 100개 이하 품목으로 시작
- 향후 데이터 증가 시 검색 최적화 필요
